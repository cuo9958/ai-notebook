# 开发路线图

## 1. 版本规划

## v0.1.0 MVP

目标：先完成“可稳定使用的本地 Markdown 笔记工具”。

范围：

- Tauri + Vue3 项目初始化
- 目录树与笔记 CRUD
- Markdown 编辑 / 预览
- 自动保存
- 自动备份
- 基础设置
- 单模型 AI 段落润色

不包含：

- 多邮箱收发
- 多模型切换
- 附件管理增强
- 云同步

## v0.2.0

目标：补全 AI 能力并开始接入邮件。

范围：

- AI 上下文增强
- AI 历史记录
- 段落备注
- 多邮箱账号配置
- 收件箱聚合与详情查看

## v0.3.0

目标：形成完整的笔记 + 邮件一体化工作台。

范围：

- 邮件发送、回复、转发
- 附件下载与打开
- 定时拉取
- 全局搜索
- 跨平台适配与安装包发布

## 2. 任务拆解

## 阶段 1：工程初始化

- 建立 `Tauri + Vue3 + TypeScript` 项目
- 引入 `Pinia`、`Vue Router`
- 配置 UI 组件库
- 搭建基础路由与主布局
- 定义目录结构与编码规范

产出：

- 可运行的桌面壳工程
- 首页、笔记页、设置页空骨架

## 阶段 2：笔记基础能力

- 实现目录树展示
- 实现笔记创建、打开、保存、删除、重命名
- 支持多级目录嵌套
- 实现标题 / 内容搜索
- 支持按创建时间、修改时间排序

产出：

- 可实际管理本地 `.md` 文件

## 阶段 3：Markdown 编辑器

- 接入编辑器组件
- 支持实时预览
- 支持代码高亮
- 支持图片插入
- 支持撤销重做
- 支持全屏预览

产出：

- 具备基础可用性的 Markdown 编辑体验

## 阶段 4：备份模块

- 实现保存即备份
- 实现备份目录配置
- 实现保留时长配置
- 实现过期清理逻辑
- 实现备份恢复能力

产出：

- 稳定的本地内容保护能力

## 阶段 5：AI 模块

- 定义统一 AI Provider 接口
- 接入首个模型
- 实现段落选中后 AI 润色
- 提取标题大纲与最近段落作为上下文
- 保存 AI 历史记录

产出：

- 可用的 AI 段落编辑流程

## 阶段 6：邮件模块

- 邮箱账号配置
- IMAP 拉取
- 统一邮件列表
- 邮件详情查看
- SMTP 发送
- 回复 / 转发 / 删除

产出：

- 基础邮件工作台

## 阶段 7：设置与发布

- 完成设置中心
- 增加主题与语言切换
- 增加日志与错误提示
- 适配三端权限差异
- 打包并验证安装包

产出：

- 可测试、可发布版本

## 3. 建议排期

### 第 1 周

- 完成工程初始化
- 完成主布局与路由
- 完成笔记目录树雏形

### 第 2 周

- 完成笔记 CRUD
- 完成 Markdown 编辑 / 预览
- 完成自动保存

### 第 3 周

- 完成自动备份
- 完成恢复与清理
- 完成设置页第一版

### 第 4 周

- 接入首个 AI 模型
- 实现段落润色
- 完成 AI 历史记录初版

### 第 5 周

- 接入邮箱账号配置
- 完成收件列表与详情

### 第 6 周

- 完成发信、回复、转发
- 兼容性测试与打包

## 4. 验收标准

### MVP 验收

- 用户可以创建目录并管理 Markdown 笔记
- 编辑内容可自动保存
- 保存后可找到对应备份
- 可对段落发起 AI 润色
- 所有配置可在设置页修改并持久化

### 完整版验收

- 支持至少 2 类主流邮箱成功收发
- 三端安装包能正常启动和使用
- 主要功能无阻塞级 bug

## 5. 下一步实施建议

如果要正式进入开发，建议按下面顺序推进：

1. 先初始化实际项目骨架
2. 先只做笔记，不要并行开发邮件
3. AI 先接一个模型跑通闭环
4. 邮件模块最后接入，降低首版复杂度
